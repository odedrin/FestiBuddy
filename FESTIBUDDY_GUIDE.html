1 What is FestiBud?

FestiBud is a harm reduction tool for people who take medications or substances and want to understand the timing, duration, and interaction of their effects. By planning ahead and visualising how multiple substances overlap across time, users can make safer decisions — avoiding dangerous combinations, preventing unintentional re-dosing, and understanding when effects will peak or subside.

The core engine is a superposition stopwatch: each substance is modelled as a rise-and-fall curve f(t), and FestiBud displays the sum of all active curves — the superposition signal — as a live graph with a deterministic prediction of the future.

FestiBud does not encourage or promote substance use. It is a safety and planning tool, designed to reduce harm for people who are already making their own informed choices. All data stays on your device — nothing is ever transmitted or stored externally.

2 The Superposition Model

The individual curve f(t)

Each stopwatch type defines a piecewise function f(t), where t is the elapsed time since the stopwatch was started. The function has four phases:

PhaseValue rangeDescription
Onset0 → onsetEndFraction × peakInitial slow ramp from zero before effects are noticeable.
ComeuponsetEndFraction × peak → peakAccelerating rise to the peak value.
Peakconstant at peakValueThe plateau. Duration is an explicit parameter.
OffsetpeakValue → 0The comedown / fade back to zero.

Each phase has its own duration (in minutes) and curve shape: linear, easeIn, easeOut, or sigmoid.

The global signal y(t)

When multiple stopwatches run simultaneously — including multiple instances of the same type started at different times — FestiBud computes:

y(t) = Σ fi(t − startTimei)

The Graph screen displays this sum. The "now" line splits it into:

  • Solid curve (past) — what the sum actually was
  • Dashed curve (future) — the deterministic prediction, assuming no new stopwatches start

Because each curve shape is fully defined by its type parameters, the future prediction is exact, not an estimate.

4.1 Stopwatches Screen

The main operational view. Shows all currently running stopwatch instances, updating 4× per second.

Header

  • Time since session start — displays the wall-clock time elapsed since the earliest currently-running stopwatch was started (HH:MM:SS). Updates every 250 ms.

Stopwatch cards

Each card shows elapsed / total time (e.g. "1h 10m / 2h") as the primary display.

  • Color stripe on the left (type color)
  • Type name and elapsed / total time — tap the ✎ edit icon to edit the start time (shifts the entire curve in place)
  • Phase progress bar — segmented bar showing all four phases. Darker segment = higher intensity. A thin vertical cursor marks the exact current position.
  • Current phase pill — shows the current phase name (Onset / Comeup / Peak / Offset).
  • ⏸ Pause / ▶ Resume button — freezes elapsed time; resume picks up from where it left off.
  • ✕ Stop button — removes the instance from the running total immediately.

Editing a start time

Tap the ✎ edit icon on any running stopwatch card. A clock picker opens showing the clock time the stopwatch started ("Started at"). Adjust the hour and minutes, then confirm. The curve shifts accordingly without restarting the stopwatch. Future times are not allowed — if the picked time is later than now, it is clamped to the current time.

Auto-cleanup

Every minute, FestiBud removes any stopwatch whose curve has fully completed (reached the end of its offset phase) more than 24 hours ago. This runs automatically in the background — no manual action needed.

Starting a stopwatch

Tap the + FAB at the bottom. A bottom sheet appears with all types, favorites first. Tap ▶ Start to launch an instance and close the sheet.

4.2 Graph Screen

Shows the full superposition graph — past (solid) and predicted future (dashed) — updating every second.

Header

Displays the wall-clock time elapsed since the earliest currently-running stopwatch was started (HH:MM:SS), labelled "elapsed".

The graph

  • Teal line (sum) — superposition y(t). Solid past, dashed future at 45% opacity.
  • Colored lines (individual) — each running stopwatch at its type color. 45% opacity past, 20% future.
  • Red dashed vertical line — "now" marker.
  • X-axis — wall-clock time. Tick density adapts to the time window.
  • Y-axis — value scale, up to ~115% of the summed peaks.

The time window spans from 20 minutes before the earliest running stopwatch to 20 minutes after the last one finishes.

Legend

The first row shows a "Sum y(t)" label for the teal superposition line. Each running stopwatch then gets its own row showing: color swatch, type name, current phase pill, and elapsed / total time (e.g. "01:10:00 / 2h 00m").

Zoom and pan

Pinch on the graph to zoom the visible time window between 1 and 12 hours. Swipe horizontally to pan backward or forward in time. A ↩ Back to Now button appears inside the graph card whenever you've panned or zoomed away from the auto window — tap it to snap back to the live view.

Plan overlay

Each saved plan has a show/hide chip below the graph card. When shown, the plan's entries appear as vertical start-time marker lines on the graph — one line per entry, labeled with the plan name and substance name. No curves are drawn; only start times are marked, keeping the live signal uncluttered.

Plan Mode button

A pill button below the graph card opens Plan Mode.

4.3 Types Screen

Manage your stopwatch type library. Two sections: Custom (your types, shown first) and Built-in (the two presets).

Type cards

Each card shows the type name, total duration, peak value, a mini sparkline curve preview, a four-phase breakdown table, and a segmented proportion bar. Action buttons:

  • ▶ Start — starts an instance directly from this screen
  • Customize / Edit — opens the type editor
  • ✕ — deletes the type (custom only; confirmation required)
  • ★ / ☆ — toggles favorite. Favorited types appear first in the add-stopwatch sheet.

4.4 Plan Mode

Accessed from the Graph screen. A separate screen (not a tab) for hypothetically placing entries and previewing the curve without starting any real stopwatches.

Example use: "If I start an Extended now and a Quick in 45 minutes, what does my evening curve look like?"

Plans

You can have multiple named plans. A chip bar at the top switches between them.

  • Tap a chip to select
  • Long-press to rename or delete
  • ✏ pencil icon on the selected chip — rename dialog
  • + New — creates a new empty plan

Plan graph

The graph previews the selected plan using vertical start-time markers for each entry — no live stopwatch curves are shown. The graph updates in real time, so as minutes pass, the markers shift correctly relative to the "now" line.

Plan entries

Each entry has a type, a human-readable time offset from now (e.g. "+2h 15m", "−30m", "now"), an Edit button (opens the time editor), and a ✕ delete button.

Tap the + FAB to add a new entry: pick a type, set the timing using any of the three anchor modes (Start at, Peak at, Comedown), tap Add Entry. The same modes are available when editing an existing entry.

Entries store an absolute target time, so a plan created today can include entries scheduled for tomorrow or later without any time drift.

5 Working with Plans

When adding or editing a plan entry, you set its timing via one of three anchor modes:

Start at

Pick the clock time at which the stopwatch begins. Offset = target time − now.

"I took it at 14:30."

Peak at

Pick when the peak phase begins. Start is back-calculated: start = peak − (onset + comeup).

"I want full effect by 19:00."

Comedown

Pick a target value (% of peak) and the clock time it should be reached during offset. Start is solved via binary search.

"25% by 17:00."

Day selector

A −1d / +1d control above the clock picker lets you shift the entry to a different day. The label shows "Today", "Tomorrow", or "+Nd" for further ahead. This makes it easy to plan across midnight without needing a full calendar UI.

The clock picker

An HH:MM picker is the main control in all three modes:

  • ▲ / ▼ — adjust hour (±1h, wraps 0–23)
  • +15 / +5 / +1 and −15 / −5 / −1 — adjust minutes (wraps 0–59)

Offset display

The offset from now is shown in a human-readable format that scales with duration:

  • < 1 hour: +45m
  • 1–24 hours: +2h 15m (minutes omitted if exactly on the hour)
  • 1+ days: +1d 3h 30m (zero components omitted)
  • Past entries use a − prefix (e.g. −30m from now)

Timeline preview

A Start → Peak → End strip below the picker shows computed wall-clock milestones, updating live as you adjust the dials.

State model

All state lives in a single AppState managed by useReducer:

interface AppState {
  types:              StopwatchType[];      // all type definitions
  activeStopwatches:  ActiveStopwatch[];    // running instances
  favoriteTypeIds:    string[];             // ordered favorite IDs
  plans:              Plan[];              // plan mode plans
}
State is persisted to AsyncStorage on every change and rehydrated on launch. Built-in types are always restored from code; custom types are merged from storage.

Curve engine — key exports

FunctionPurpose
evaluate(type, elapsedMs)Returns f(t). Called 300× per graph render and 4×/s per card.
totalDuration(type)Sum of all four phase durations.
currentPhase(type, elapsedMs)Returns 'onset' | 'comeup' | 'peak' | 'offset' | 'done' | 'pending'.
solveStartForPeakAt(type, peakClockMs)Returns startTime such that peak begins at peakClockMs.
solveStartForOffsetValue(type, value, clockMs)64-iteration binary search. Returns startTime such that the offset phase hits value at clockMs. Accurate to sub-ms.
formatDuration(ms)"1h 20m" — used in type cards and plan entries.
formatElapsed(ms)"01:23:45" — used in stopwatch cards and graph header.

Graph component

Graph accepts two optional override props:

  • overrideEntries?: GraphEntry[] — when provided, renders these entries instead of the live active stopwatches. Pass [] to suppress all curves (Plan Mode does this so only markers are visible).
  • planMarkers?: PlanMarker[] — renders vertical start-time marker lines on the graph. Each marker shows a full-height colored line with a label chip ("Plan name · Substance name").
interface GraphEntry {
  type:      StopwatchType;
  startTime: number;  // absolute ms
}

interface PlanMarker {
  startTime: number;  // absolute ms
  label:     string;  // e.g. "Festival · Extended"
  color:     string;  // CSS/RN color string
}

Paths are built from 300 sample points, split at "now" into solid (past) and dashed (future) SVG <Path> elements.

PlannedEntry model

Plan entries store an absolute timestamp rather than a relative offset:

interface PlannedEntry {
  id:         string;
  typeId:     string;
  targetTime: number;  // absolute ms (was: offsetMinutes)
}

On hydration, older persisted entries with offsetMinutes are automatically migrated: targetTime = Date.now() + offsetMinutes × 60 000.

formatOffset utility

utils/formatOffset.ts — converts a millisecond delta to a compact human-readable string used throughout plan UI:

formatOffset(diffMs: number): string
// 0       → "now"
// 45 min  → "+45m"
// 2h 15m  → "+2h 15m"
// 1d 3h   → "+1d 3h"
// negative → "−" prefix

8 Limitations & Roadmap

Priority roadmap (next up)

  1. 1
    Substance database

    Built-in read-only library of widely used substances with pharmacokinetic data, bundled statically in the app (no runtime API calls — fully offline). Data sources: PsychonautWiki GraphQL API (onset, comeup, peak, offset durations per route of administration) merged with TripSit combos.json (interaction pairs and severity). Duration ranges from PsychonautWiki are string values (e.g. "30–60 minutes") — midpoints are used as default curve parameters, which users can override via the type editor. Interaction severity levels map directly to TripSit's: Dangerous / Unsafe / Caution / Low Risk.

  2. 2
    Graph zoom and pan are not working smoothly

    Should be fixed

  3. 3
    Interaction warnings

    When multiple substances are active or planned, surface known interaction pairs with severity (Dangerous / Caution / Low Risk / Synergy) and a brief clinical mechanism note. Warnings are ON by default; users can opt out in settings.

  4. 4
    Settings screen

    A dedicated settings screen for user preferences. Initial scope: opt-out toggle for interaction warnings, notification permissions and preferences (plan start-time alerts).

Current limitations

No history

Stopped stopwatches are simply removed — no log of what ran and when.

Potential future features

History log

Record of all past runs: start time, stop time, integrated area under the curve.

Notifications

Notify the user when a plan entry's scheduled start time arrives. No phase-transition or threshold notifications.

Type library sync

Backup and restore custom types across devices.
