FestiBud

User Guide & Product Documentation

v1.2 iOS / Android React Native + Expo Fully offline June 2026

1 What is FestiBud?

FestiBud is a harm reduction tool for people who take medications or substances and want to understand the timing, duration, and interaction of their effects. By planning ahead and visualising how multiple substances overlap across time, users can make safer decisions — avoiding dangerous combinations, preventing unintentional re-dosing, and understanding when effects will peak or subside.

The core engine is a superposition stopwatch: each substance is modelled as a rise-and-fall curve f(t), and FestiBud displays the sum of all active curves — the superposition signal — as a live graph with a deterministic prediction of the future.

FestiBud does not encourage or promote substance use. It is a safety and planning tool, designed to reduce harm for people who are already making their own informed choices. All data stays on your device — nothing is ever transmitted or stored externally.

2 The Superposition Model

The individual curve f(t)

Each stopwatch type defines a piecewise function f(t), where t is the elapsed time since the stopwatch was started. The function has four phases:

PhaseValue rangeDescription
Onset0 → onsetEndFraction × peakInitial slow ramp from zero before effects are noticeable.
ComeuponsetEndFraction × peak → peakAccelerating rise to the peak value.
Peakconstant at peakValueThe plateau. Duration is an explicit parameter.
OffsetpeakValue → 0The comedown / fade back to zero.

Each phase has its own duration (in minutes) and curve shape: linear, easeIn, easeOut, or sigmoid.

The global signal y(t)

When multiple stopwatches run simultaneously — including multiple instances of the same type started at different times — FestiBud computes:

y(t) = Σ fi(t − startTimei)

The Graph screen displays this sum. The "now" line splits it into:

  • Solid curve (past) — what the sum actually was
  • Dashed curve (future) — the deterministic prediction, assuming no new stopwatches start

Because each curve shape is fully defined by its type parameters, the future prediction is exact, not an estimate.

Substance types

A third category — substance types — ships bundled with the app. These are read-only profiles derived from PsychonautWiki pharmacokinetic data (oral ROA midpoints unless otherwise noted). They behave like built-in types in that they cannot be deleted or edited, but they are displayed separately under a Substances section in both the Types screen and the Add Stopwatch sheet.

The 16 bundled substances are:

  • Empathogens: MDMA, MDA
  • Psychedelics: LSD, Psilocybin, 2C-B, Mescaline, DMT (smoked)
  • Dissociatives: Ketamine
  • Depressants: Alcohol (1 drink), GHB
  • Stimulants: Cocaine, Amphetamine, Caffeine, Nicotine (smoked)
  • Cannabis: Cannabis (smoked), Cannabis (edible)

Substance types are never stored in AsyncStorage — they are always merged from code at hydration time so that bundled data updates automatically when the app is updated.

Custom types

Custom types appear above built-in types in the Types screen and can be edited or deleted at any time. Create one from scratch via the editor, or tap Customize on any built-in to duplicate it.

4.1 Stopwatches Screen

The main operational view. Shows all currently running stopwatch instances, updating 4× per second.

Header

  • Time since session start — displays the wall-clock time elapsed since the earliest currently-running stopwatch was started (HH:MM:SS). Updates every 250 ms.

Stopwatch cards

Each card shows elapsed / total time (e.g. "1h 10m / 2h") as the primary display.

  • Color stripe on the left (type color)
  • Type name and elapsed / total time — tap the ✎ edit icon to edit the start time (shifts the entire curve in place)
  • Phase progress bar — segmented bar showing all four phases. Darker segment = higher intensity. A thin vertical cursor marks the exact current position.
  • Current phase pill — shows the current phase name (Onset / Comeup / Peak / Offset).
  • ⏸ Pause / ▶ Resume button — freezes elapsed time; resume picks up from where it left off.
  • ✕ Stop button — removes the instance from the running total immediately.

Editing a start time

Tap the ✎ edit icon on any running stopwatch card. A clock picker opens showing the clock time the stopwatch started ("Started at"). Adjust the hour and minutes, then confirm. The curve shifts accordingly without restarting the stopwatch. Future times are not allowed — if the picked time is later than now, it is clamped to the current time.

Auto-cleanup

Every minute, FestiBud removes any stopwatch whose curve has fully completed (reached the end of its offset phase) more than 24 hours ago. This runs automatically in the background — no manual action needed.

Starting a stopwatch

Tap the + FAB at the bottom. A bottom sheet appears with all types, favorites first, followed by general types and a Substances section at the bottom. Tap ▶ Start to launch an instance and close the sheet. Each substance row also shows a ↗ button that opens the substance's PsychonautWiki article in the system browser without closing the sheet.

When starting a substance type while other substance stopwatches are already running, FestiBud checks for known drug interactions. If any are found and Interaction Warnings are enabled in Settings, a modal appears listing the interaction pairs with severity badges before the stopwatch starts. You can Start anyway or Cancel to go back to the sheet.

4.2 Graph Screen

Shows the full superposition graph — past (solid) and predicted future (dashed) — updating every second.

Header

Displays the wall-clock time elapsed since the earliest currently-running stopwatch was started (HH:MM:SS), labelled "elapsed".

The graph

  • Teal line (sum) — superposition y(t). Solid past, dashed future at 45% opacity.
  • Colored lines (individual) — each running stopwatch at its type color. 45% opacity past, 20% future.
  • Red dashed vertical line — "now" marker.
  • X-axis — wall-clock time. Tick density adapts to the time window.
  • Y-axis — value scale, up to ~115% of the summed peaks.

The time window spans from 20 minutes before the earliest running stopwatch to 20 minutes after the last one finishes.

Legend

The first row shows a "Sum y(t)" label for the teal superposition line. Each running stopwatch then gets its own row showing: color swatch, type name, current phase pill, and elapsed / total time (e.g. "01:10:00 / 2h 00m").

Zoom and pan

Pinch on the graph to zoom the visible time window between 1 and 12 hours. Swipe horizontally to pan backward or forward in time. A ↩ Back to Now button appears inside the graph card whenever you've panned or zoomed away from the auto window — tap it to snap back to the live view.

Plan overlay

Each saved plan has a show/hide chip below the graph card. When shown, the plan's entries appear as vertical start-time marker lines on the graph — one line per entry, labeled with the plan name and substance name. No curves are drawn; only start times are marked, keeping the live signal uncluttered.

Plan Mode button

A pill button below the graph card opens Plan Mode.

4.3 Types Screen

Manage your stopwatch type library. Sections: Custom (your types, shown first), Built-in (the two presets), and Substances. Types you've hidden appear in a collapsible Hidden section at the bottom.

Type cards

Each card shows the type name, a mini sparkline curve preview, total duration, a four-phase breakdown table (duration and proportion per phase), and a segmented proportion bar. Action buttons:

  • ▶ Start — starts an instance directly from this screen. Triggers a haptic tap on confirmation. If the type is a substance and interaction warnings are enabled, an interaction check runs first.
  • Customize / Edit — opens the type editor
  • Hide — moves the type card to the collapsible Hidden section at the bottom of the screen. The type still works normally; it just de-clutters the main list.
  • ✕ — deletes the type (custom only; confirmation required)
  • ★ / ☆ — toggles favorite. Favorited types appear first in the add-stopwatch sheet.
  • ↗ — (substance types only) opens the substance's PsychonautWiki article in the system browser.

If any instances of a type are currently running, a small count badge (pill in the type's color) appears next to the name.

Hiding a type

Tap Hide on any type card to remove it from its section without deleting it. The type keeps all its settings and can still be started from the Add Stopwatch sheet. A collapsible Hidden (n) section appears at the bottom of the Types screen. Expand it to see all hidden types; each card shows an Unhide button that returns the type to its original section. Hidden state is persisted in AsyncStorage.

4.4 Plan Mode

Accessed from the Graph screen. A separate screen (not a tab) for hypothetically placing entries and previewing the curve without starting any real stopwatches.

Example use: "If I start an Extended now and a Quick in 45 minutes, what does my evening curve look like?"

Plans

You can have multiple named plans. A chip bar at the top switches between them.

  • Tap a chip to select
  • Long-press to rename or delete
  • ✏ pencil icon on the selected chip — rename dialog
  • + New — creates a new empty plan

Plan graph

The graph previews the selected plan using vertical start-time markers for each entry — no live stopwatch curves are shown. The graph updates in real time, so as minutes pass, the markers shift correctly relative to the "now" line.

Plan entries

Each entry has a type, a human-readable time offset from now (e.g. "+2h 15m", "−30m", "now"), an Edit button (opens the time editor), and a ✕ delete button.

Tap the + FAB to add a new entry: pick a type, set the timing using any of the three anchor modes (Start at, Peak at, Comedown), tap Add Entry. The same modes are available when editing an existing entry.

When adding a substance type that would interact with substances already in the plan, an interaction warning popup appears (if warnings are enabled) before the entry is committed. Tap Add anyway to proceed or Cancel to discard.

Entries store an absolute target time, so a plan created today can include entries scheduled for tomorrow or later without any time drift.

Persistent interaction warnings

Below the entry list, FestiBud shows a persistent Interactions panel listing all known interaction pairs between substance entries currently in the plan — sorted by severity (Dangerous first). Each row shows a severity badge and the two substance names; tap a row to expand the clinical note. This panel only appears when Interaction Warnings are enabled in Settings and the plan contains at least two interacting substances.

4.5 Substance Database

The Types screen has a third section — Substances — showing the 16 read-only profiles bundled with the app. These cannot be edited or deleted, but you can copy one via Customize to create an editable variant.

4.6 Drug Interactions

FestiBud bundles a curated interaction database based on TripSit combos data, covering approximately 35 substance pairs. When you combine substances that have a known interaction, FestiBud warns you before you commit — either at stopwatch start or plan entry add.

Severity levels

StatusColorMeaning
☠ DangerousRedLife-threatening risk. Avoid. (e.g. Alcohol + GHB, GHB + Ketamine)
⚠ UnsafeOrangeSignificant risk of serious harm. (e.g. GHB + MDMA, Amphetamine + Cocaine)
⚡ CautionYellowUnpredictable interaction; use care. (e.g. Alcohol + Cocaine)
↓ Low Risk & DecreaseGreenGenerally manageable; one substance reduces the other's effects.
✦ Low Risk & SynergyGreenGenerally manageable; effects combine in a predictable way.
• Low Risk & No SynergyGreenMinimal interaction expected.

Warning popup

When you tap ▶ Start on a substance that interacts with a currently active substance stopwatch, or when you tap Add Entry in Plan Mode with a substance that interacts with entries already in the plan, a blocking modal appears showing:

  • The interaction pairs (sorted by severity, most severe first)
  • A severity badge per pair
  • The two substance names
  • An expandable clinical note (tap a row to reveal)

Two buttons: Cancel (go back without starting/adding) and Start anyway / Add anyway (proceed).

The popup only appears when Interaction Warnings is enabled in Settings (on by default). Disable it there to suppress all interaction popups and persistent panels.

Interaction data covers substance-to-substance risks only between the 16 bundled profiles. Interactions with unlisted substances or user-created custom types are not checked.

4.7 Settings

Accessed via the gear icon tab. Contains app-wide preferences and attribution.

Appearance

  • Color Scheme — a three-way segmented control: System (default, follows iOS/Android), Light, or Dark. The selection is saved immediately to AsyncStorage and applied across all screens — including the tab bar, graph, interaction modals, and all card backgrounds. Graph axis/grid colors, label contrast, and the tab bar background all adapt automatically.

Harm Reduction

  • Interaction Warnings — toggle (on by default). When enabled, FestiBud shows warning popups before starting a substance stopwatch or adding a substance plan entry that creates a known interaction, and renders the persistent interactions panel in Plan Mode. Disable to remove all interaction UI.

About

Attribution rows showing the data sources used in the bundled substance database. The first two rows are tappable links that open in the system browser:

  • PK data source — PsychonautWiki ↗ — opens https://psychonautwiki.org
  • Interaction data — TripSit combos ↗ — opens https://tripsit.me
  • Substances — 16 bundled profiles
  • Privacy & disclosure — View ↗ — opens the Disclosure screen (see below)

Disclosure screen

A full-screen page sheet summarising FestiBud's privacy stance and harm reduction context. It contains three cards:

  • 100% offline · no data leaves your device — confirms that everything is stored locally and nothing is transmitted externally.
  • For harm reduction reference only — explains that PK data are population midpoints, individual responses vary significantly, and the app is not medical advice.
  • Interaction warnings — explains the TripSit-sourced interaction data and notes that the absence of a warning does not imply safety.

First launch: the Disclosure screen appears automatically on the first time the app is opened. A Don't show again checkbox (checked by default) is shown above the Continue button. If left checked, a flag is written to AsyncStorage and the screen will not appear automatically on subsequent launches. Unchecking it and tapping Continue dismisses the screen without writing the flag, so it will appear again next launch.

The screen can always be reopened via Settings → Privacy & disclosure → View ↗. In this context the checkbox is absent; a small Show on next launch link at the bottom clears the suppression flag so the screen will reappear automatically on the next cold start.

Disclaimer

A reminder that FestiBud provides pharmacokinetic reference data for harm reduction purposes only — duration estimates are population midpoints, individual responses vary, and a medical professional should be consulted if in doubt.

5 Working with Plans

When adding or editing a plan entry, you set its timing via one of three anchor modes:

Start at

Pick the clock time at which the stopwatch begins. Offset = target time − now.

"I took it at 14:30."

Peak at

Pick when the peak phase begins. Start is back-calculated: start = peak − (onset + comeup).

"I want full effect by 19:00."

Comedown

Pick a target value (% of peak) and the clock time it should be reached during offset. Start is solved via binary search.

"25% by 17:00."

Day selector

A −1d / +1d control above the clock picker lets you shift the entry to a different day. The label shows "Today", "Tomorrow", or "+Nd" for further ahead. This makes it easy to plan across midnight without needing a full calendar UI.

The clock picker

An HH:MM picker is the main control in all three modes:

  • ▲ / ▼ — adjust hour (±1h, wraps 0–23)
  • +15 / +5 / +1 and −15 / −5 / −1 — adjust minutes (wraps 0–59)

Offset display

The offset from now is shown in a human-readable format that scales with duration:

  • < 1 hour: +45m
  • 1–24 hours: +2h 15m (minutes omitted if exactly on the hour)
  • 1+ days: +1d 3h 30m (zero components omitted)
  • Past entries use a − prefix (e.g. −30m from now)

Timeline preview

A Start → Peak → End strip below the picker shows computed wall-clock milestones, updating live as you adjust the dials.

State model

All state lives in a single AppState managed by useReducer:

interface AppState {
  types:                   StopwatchType[];   // all type definitions (built-in + substance + custom)
  activeStopwatches:       ActiveStopwatch[]; // running instances
  favoriteTypeIds:         string[];          // ordered favorite IDs
  plans:                   Plan[];            // plan mode plans
  showInteractionWarnings: boolean;           // toggle for interaction warning UI (default: true)
}
State is persisted to AsyncStorage on every change and rehydrated on launch. Built-in types (isBuiltIn: true) and substance types (isSubstance: true) are always restored from code at hydration and are never saved to AsyncStorage — only custom types are merged from storage. This means bundled data updates automatically when the app version is updated.

Graph component

Graph accepts two optional override props:

  • overrideEntries?: GraphEntry[] — when provided, renders these entries instead of the live active stopwatches. Pass [] to suppress all curves (Plan Mode does this so only markers are visible).
  • planMarkers?: PlanMarker[] — renders vertical start-time marker lines on the graph. Each marker shows a full-height colored line with a label chip ("Plan name · Substance name").
interface GraphEntry {
  type:      StopwatchType;
  startTime: number;  // absolute ms
}

interface PlanMarker {
  startTime: number;  // absolute ms
  label:     string;  // e.g. "Festival · Extended"
  color:     string;  // CSS/RN color string
}

Paths are built from 150 sample points, split at "now" into solid (past) and dashed (future) SVG <Path> elements.

PlannedEntry model

Plan entries store an absolute timestamp rather than a relative offset:

interface PlannedEntry {
  id:         string;
  typeId:     string;
  targetTime: number;  // absolute ms (was: offsetMinutes)
}

On hydration, older persisted entries with offsetMinutes are automatically migrated: targetTime = Date.now() + offsetMinutes × 60 000.

formatOffset utility

utils/formatOffset.ts — converts a millisecond delta to a compact human-readable string used throughout plan UI:

formatOffset(diffMs: number): string
// 0       → "now"
// 45 min  → "+45m"
// 2h 15m  → "+2h 15m"
// 1d 3h   → "+1d 3h"
// negative → "−" prefix

8 Limitations & Roadmap

Priority roadmap (next up)

  1. 1
    iOS App Store release

    Submit to App Store under Health & Fitness, age rating 17+. Prepare screenshots, metadata, and privacy disclosure (no data leaves the device). See Section 9 for submission strategy.

  2. 2
    Notifications

    Notify the user when a plan entry's scheduled start time arrives. No phase-transition or threshold notifications. Opt-out available in Settings.

  3. 3
    Merge Stopwatches + Graph into one screen; promote Plan Mode to a tab

    Currently the graph lives on a separate tab and Plan Mode is a pushed stack screen. Consider collapsing Stopwatches and Graph into a single scrollable or split view, and giving Plan Mode its own permanent tab — reducing navigation depth and making planning a first-class destination.

No history

Stopped stopwatches are simply removed — no log of what ran and when.

Potential future features

History log

Record of all past runs: start time, stop time, integrated area under the curve.

Notifications

Notify the user when a plan entry's scheduled start time arrives. No phase-transition or threshold notifications.

Type library sync

Backup and restore custom types across devices.

Emergency numbers by country

A dedicated screen showing the relevant emergency and poison-control numbers for your country. Searchable/filterable country dropdown. Numbers open the phone dialer directly. Intended as a quick-access safety reference — not a replacement for calling emergency services.

Reagent testing guide

Per-substance reference showing which reagents to use (Marquis, Mecke, Simon's, etc.) and the expected color reactions for a positive result. Helps users verify substance identity before consumption. Sourced from published harm-reduction references (DanceSafe, Reagent.co.uk).

Drag-to-reorder types

Allow users to reorder stopwatch types within each section (Custom, Built-in, Substances) via drag handles. Order is persisted across sessions. Currently disabled pending a reliable cross-session implementation.
