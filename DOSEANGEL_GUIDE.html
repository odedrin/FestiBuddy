1 What is DoseAngel?

DoseAngel is a harm reduction tool for people who take substances and want to understand the timing, duration, and interaction of their effects. By planning ahead and visualising how multiple substances overlap across time, users can make safer decisions — avoiding dangerous combinations, preventing unintentional re-dosing, and understanding when effects will peak or subside.

The core engine is a superposition stopwatch: each substance is modelled as a rise-and-fall curve f(t), and DoseAngel displays the sum of all active curves — the superposition signal — as a live graph with a deterministic prediction of the future.

DoseAngel does not encourage or promote substance use. It is a safety and planning tool, designed to reduce harm for people who are already making their own informed choices. All data stays on your device — nothing is ever transmitted or stored externally.

2 The Superposition Model

The individual curve f(t)

Each stopwatch type defines a piecewise function f(t), where t is the elapsed time since the stopwatch was started. The function has four phases:

PhaseValue rangeDescription
Onset0 → onsetEndFraction × peakInitial slow ramp from zero before effects are noticeable.
ComeuponsetEndFraction × peak → peakAccelerating rise to the peak value.
Peakconstant at peakValueThe plateau. Duration is an explicit parameter.
OffsetpeakValue → 0The comedown / fade back to zero.

Each phase has its own duration (in minutes) and curve shape: linear, easeIn, easeOut, or sigmoid.

The global signal y(t)

When multiple stopwatches run simultaneously — including multiple instances of the same type started at different times — DoseAngel computes:

y(t) = Σ fi(t − startTimei)

The Graph screen displays this sum. The "now" line splits it into:

  • Solid curve (past) — what the sum actually was
  • Dashed curve (future) — the deterministic prediction, assuming no new stopwatches start

Because each curve shape is fully defined by its type parameters, the future prediction is exact, not an estimate.

Substance types

A third category — substance types — ships bundled with the app. These are read-only profiles derived from PsychonautWiki pharmacokinetic data (oral ROA midpoints unless otherwise noted). They behave like built-in types in that they cannot be deleted or edited, but they are displayed separately under a Substances section in both the Substances Library (Settings → Manage) and the Add Stopwatch sheet.

The 16 bundled substances are:

  • Empathogens: MDMA, MDA
  • Psychedelics: LSD, Psilocybin, 2C-B, Mescaline, DMT (smoked)
  • Dissociatives: Ketamine
  • Depressants: Alcohol (1 drink), GHB
  • Stimulants: Cocaine, Amphetamine, Caffeine, Nicotine (smoked)
  • Cannabis: Cannabis (smoked), Cannabis (edible)

Substance types are never stored in AsyncStorage — they are always merged from code at hydration time so that bundled data updates automatically when the app is updated.

Custom types

Custom types appear above built-in types in the Substances Library and can be edited or deleted at any time. Create one from scratch via the editor, or tap Customize on any built-in to duplicate it.

4.1 Live Screen

The main operational view. Shows the full superposition graph alongside a scrollable legend of all running stopwatches, updating every second. A bold "Live" headline and the subtitle "The current state of your trip" sit at the top of the screen so its purpose is clear at a glance. The whole screen is a vertical scroll view, so expanding the legend (see Container 3) never has to shrink the graph — it just pushes below the fold and you scroll to see it.

Container 1 — Overlay plans

Sits at the top of the screen, above the graph. Only shown when at least one plan exists. A horizontal row of chips — one per saved plan — lets you toggle any plan's entries onto the graph without navigating away. The overlay mode (Markers or Curves) is controlled in Settings → Plan overlay:

  • Markers (default) — a vertical start-time line per entry, labeled with plan and substance name.
  • Curves — draws the full predicted curve for each plan entry at its type color, overlaid on the live graph.

Container 2 — Graph card

A compact elapsed timer sits above the graph: an "ELAPSED" label, the wall-clock time since the earliest running stopwatch (HH:MM:SS), and a "since first dose" suffix.

  • Teal line (sum) — superposition y(t). Solid past, dashed future at 45% opacity.
  • Colored lines (individual) — each running stopwatch at its type color. 45% opacity past, 20% future.
  • Red dashed vertical line — "now" marker.
  • X-axis — wall-clock time. Tick density adapts to the time window.
  • Y-axis — value scale, up to ~115% of the summed peaks.

Pinch to zoom (1–24 h window). Swipe horizontally to pan.

When you pan or zoom so that one or more items (running stopwatches or overlaid plan entries) are entirely outside the visible window, off-screen indicator pills appear on the relevant edge of the graph card: ‹ N on the left if N items have already ended before the left edge, and N › on the right if N items haven't started yet relative to the right edge. Tapping the Now or Day nav buttons typically brings all items back into view and the pills disappear.

A navigation bar sits below the graph inside the card with six controls:

  • « ‹ — pan backward by 100% / 50% of the current window width
  • Now — re-centers the view on the current time. If your current zoom is 12 h or less it is preserved; if you are in the 24 h day view it resets to a 6 h window so the view actually changes. Highlights (teal) when you have panned away from center.
  • Day — switches to a 24 h window centered on the current time (12 h past, 12 h future). Highlights (teal) when you are not in the 24 h view — press it to enter day view, it dims when day view is active.
  • › » — pan forward by 50% / 100% of the current window width

Container 3 — Running now (legend)

A collapsible card below the graph. Tapping its header (or the chevron) toggles it open/closed; the header always shows the title ("Running now · N" or "N selected") regardless of collapsed state. Collapsing it never resizes the graph — it only hides the stopwatch rows. When expanded, the row list is a plain (non-scrolling) view, so if it's taller than the screen you scroll the whole page to reach the rest of the list rather than the legend scrolling internally. When expanded, action buttons also appear in the header:

  • Select / Cancel — enters multi-select mode (visible when at least one stopwatch is running). Long-pressing any row also enters select mode and selects that row. In select mode a Remove button appears; tapping it shows a confirmation prompt before deleting all selected stopwatches.

Each row in the legend shows:

  • Color stripe on the left edge
  • Type name and current phase pill
  • Elapsed / total time (e.g. "01:10:00 / 02:00:00")
  • Phase progress bar — compact 5 px segmented bar (onset → comeup → peak → offset, each at a different opacity). A thin vertical cursor marks the exact current position.
  • ✎ edit button — opens the drum-roll clock picker to adjust the start time
  • ✕ stop button — removes the stopwatch (confirmation required)
  • In select mode, a checkbox circle replaces the action buttons

+ FAB

A floating action button in the bottom-right corner, fixed in place regardless of scroll position, opens the Add Stopwatch bottom sheet. It lists all types (favorites first, then general, then Substances), may show colored interaction badges for known interactions with active stopwatches (see §4.5 for badge icons), and includes ↗ links to PsychonautWiki for substance types. Tap ▶ Start to launch an instance. If the type is a substance with a Caution, Unsafe, or Dangerous interaction and Warning Popups are enabled, a blocking confirmation modal appears first.

4.2 Plan Screen

A permanent tab for hypothetically placing entries and previewing the curve without starting any real stopwatches.

Example use: "If I start an Extended now and a Quick in 45 minutes, what does my evening curve look like?"

Layout

The screen headline ("Plan" + a one-line description) and the chip bar are fixed at the top. Everything below scrolls in a single outer ScrollView — the same pattern used on the Live screen — so collapsing a card never shrinks the graph; the page scrolls to reveal content instead:

  1. Headline (fixed) — screen title and a short explanation of what the screen is for.
  2. Chip bar (fixed) — switches between saved plans. Horizontal scroll if many plans exist.
  3. Graph card (fixed height, scrolls with the page) — previews the selected plan's curve.
  4. Entry card (collapsible) — list of plan entries, with an ✏ Edit button in its header.
  5. Interactions card (collapsible, only shown when interactions exist) — full list of interaction pairs for the current plan.

Managing plans

The chip bar at the top lists all saved plans.

  • Tap a chip to select it
  • Long-press for rename / delete options (opens the same rename dialog used previously)
  • + New — opens the New Plan popup (see below) instead of creating an empty plan instantly

Creating a plan

Tapping + New opens a New Plan sheet covering most of the screen:

  1. Enter a name, or keep the suggested default (e.g. "Plan 3").
  2. Tap + Add substance to open the familiar type list + anchor-mode time picker (Start at / Peak at / Comedown). Confirming stages it into a list within the sheet — nothing is saved to the plan yet.
  3. Repeat to add as many substances as needed, including the same substance more than once (e.g. planning a redose).
  4. Each staged row has its own Edit (reopens the time picker) and ✕ (remove) button, so the whole plan can be adjusted before committing.
  5. Adding a substance that interacts with one already staged triggers the same blocking interaction warning popup described below.
  6. Tap Done to create the plan with everything staged, or Cancel to discard the draft entirely.

Editing a plan

The ✏ Edit button in the entry card's header (not tucked into the chip bar) opens an Edit Plan sheet — the same staged add/edit/delete-substance UI as New Plan, pre-filled with the selected plan's current name and entries:

  • Rename the plan via the name field.
  • Add, edit (time), or delete substances the same way as when creating a plan.
  • Tap Save to commit the name and entries in one update, or Cancel to discard changes.
  • A Delete Plan action sits at the bottom of the sheet (closes the sheet immediately, then runs the same confirmation as long-pressing a chip).

This is the primary way to add substances to an existing plan — there is no more "+" FAB on the Plan screen.

Plan graph

The graph previews the selected plan using vertical start-time markers for each entry — no live stopwatch curves are shown. The graph updates in real time so markers shift correctly as minutes pass. It renders at a fixed height (matching the Live screen's graph card) since the page around it scrolls rather than the card flexing to fill space.

The same off-screen indicator pills as on the Live screen appear here: ‹ N on the left edge when N plan entries have already ended before the visible window, and N › on the right when N entries haven't started yet relative to the right edge.

The same navigation bar as on the Live screen appears below the plan graph — «, ‹, Now, Day, ›, » — with identical pan/zoom behaviour and highlight logic.

Plan entries

The entry card's header shows the plan name, an entry count, and a chevron (▾/▸) — tap anywhere in the header to collapse or expand the card. Each entry shows: type color dot, type name, duration, and the planned wall-clock time (e.g. "Today 22:30"). Action buttons: Edit (opens the time editor) and ✕ (deletes the entry). When expanded, the list is not independently scrollable — it renders at full height and the outer page scrolls, so every entry is reachable without a cramped inner scroll box.

When adding a substance type that would interact with substances already in the plan, an interaction warning popup appears (if Warning Popups are enabled in Settings) before the entry is committed. Tap Add anyway to proceed or Cancel to discard.

Entries store an absolute target time, so a plan created today can include entries scheduled for tomorrow or later without drift.

Persistent interaction warnings

A dedicated, collapsible Interactions card below the entry list shows all known interaction pairs between substance entries in the plan — sorted by severity (Dangerous first). Its header shows a pair count and a chevron, same as the entry card. Each row shows a severity badge and the two substance names; tap to expand the clinical note. This card only appears when Warning Popups are enabled in Settings and the plan contains at least two interacting substances.

4.3 Combos Screen

A permanent tab (labelled Combos in the tab bar) dedicated to checking substance combinations. On-screen it's titled "Check a Combo", with a one-line description underneath — the same headline treatment as the Live and Plan screens.

This is deliberately its own tab rather than a page tucked inside Settings or a searchable list of every pair: it's the app's harm-reduction reference of record, so it needs to stay one tap away regardless of what else is going on.

Picking a combination

All 16 bundled substances appear as a wrapped grid of chips, each tinted with the substance's own color. The flow:

  1. Tap a substance to select it — its chip fills solid. Every other substance with known interaction data against it gets a small corner badge, using the same icon set as elsewhere (⚠ Dangerous/Unsafe/Caution, ↑ Synergy, ↓ Decrease, ▲ No Synergy), so you can see at a glance which second pick will actually show something.
  2. Tap a second substance. Both chips highlight, and an explanation card pinned to the bottom of the screen shows the severity badge, both substance names (each with a ⓘ link to PsychonautWiki), and the full clinical note. If no data exists for that specific pair, the card instead reads "No documented interaction" with a reminder that this doesn't mean the combination is safe.
  3. Only two substances can be selected at once. While two are picked, every other chip dims and stops responding to taps — tap one of the two selected chips again to deselect it before picking a different combination.
Badges on this screen are shown unconditionally, even if Interaction Badges is turned off in Settings. That toggle only controls the incidental badges in the Add Stopwatch sheet and Substances Library — Combos exists specifically to check combinations, so it never hides that information.

Quick access from Live

When 2 or more substances are currently running, a small ⚠ Check Combo button appears in the "Running now" card header on the Live screen. Tapping it jumps straight to the Combos tab with those substances pre-selected, so the explanation card is visible immediately.

4.5 Drug Interactions

DoseAngel bundles a curated interaction database based on TripSit combos data, covering approximately 35 substance pairs. When you combine substances that have a known interaction, DoseAngel warns you before you commit — either at stopwatch start or plan entry add. To browse or look up a combination directly, see the Combos tab (§4.3).

Severity levels

StatusColorMeaning
☠ DangerousRedLife-threatening risk. Avoid. (e.g. Alcohol + GHB, GHB + Ketamine)
⚠ UnsafeOrangeSignificant risk of serious harm. (e.g. GHB + MDMA, Amphetamine + Cocaine)
⚡ CautionYellowUnpredictable interaction; use care. (e.g. Alcohol + Cocaine)
↑ Low Risk & SynergyGreenGenerally manageable; effects amplify each other in a predictable way.
↓ Low Risk & DecreaseGreenGenerally manageable; one substance reduces the other's effects.
▲ Low Risk & No SynergyGreenMinimal interaction expected.

Warning popup

When you tap ▶ Start on a substance that interacts with a currently active substance stopwatch, or when you tap Add Entry in Plan Mode with a substance that interacts with entries already in the plan, a blocking modal appears showing:

  • The interaction pairs (sorted by severity, most severe first)
  • A severity badge per pair
  • The two substance names
  • An expandable clinical note (tap a row to reveal)

Two buttons: Cancel (go back without starting/adding) and Start anyway / Add anyway (proceed).

The popup only fires for Caution, Unsafe, and Dangerous interactions. Low Risk combinations (Synergy, Decrease, No Synergy) are intentionally excluded — they still appear as badges in the type list, but never block you with a popup.

The popup only appears when Warning Popups is enabled in Settings (on by default). A separate Interaction Badges toggle controls the badges shown in the type list before you tap Start — the two settings are independent so you can show badges without popups, or vice versa.

Interaction data covers substance-to-substance risks only between the 16 bundled profiles. Interactions with unlisted substances or user-created custom types are not checked.

4.6 Settings

Accessed via the gear icon tab. Contains app-wide preferences, the Substances Library, and attribution.

Appearance

  • Color Scheme — a three-way segmented control: System (default, follows iOS/Android), Light, or Dark. The selection is saved immediately to AsyncStorage and applied across all screens — including the tab bar, graph, interaction modals, and all card backgrounds. Graph axis/grid colors, label contrast, and the tab bar background all adapt automatically.

Harm Reduction

  • Interaction Badges — toggle (on by default). When enabled, a colored badge appears next to any substance in the Add Stopwatch or Add Plan Entry sheet when that substance has a known interaction with a currently active stopwatch or existing plan entry. The badge icon reflects the interaction type: ⚠ Dangerous/Unsafe/Caution, ↑ Low Risk & Synergy, ↓ Low Risk & Decrease, ▲ Low Risk & No Synergy. When a substance has multiple simultaneous interactions, the most important one is shown (Synergy takes priority over Decrease, which takes priority over No Synergy). Disable to hide all badges while keeping warning popups active.
  • Warning Popups — toggle (on by default). When enabled, a blocking confirmation modal appears before starting a substance stopwatch or adding a plan entry that creates a known interaction. Also controls the persistent Interactions panel at the bottom of the Plan screen. Disable to proceed silently without any blocking dialog.

Substances Library

Formerly a standalone Types tab — now a collapsible section embedded directly in Settings, collapsed by default behind a single Manage row that shows a live count of configured types (e.g. "12 types configured"). Tapping the row, or its ▼ / ▲ chevron, expands the full type-management list in place; there is no separate screen to navigate to.

While expanded, the Manage row is sticky: as you scroll down through the type list, the row stays pinned to the top of the screen so it can be collapsed again from anywhere without first scrolling back up. Collapsing it returns Settings to its normal single-row footprint.

Expanded sections, in order: Custom (your types, shown first), Built-in (the two presets), and Substances. Types you've hidden appear in a nested, non-sticky collapsible Hidden section at the bottom — independent of the outer sticky toggle.

Type cards. Each card shows the type name, a mini sparkline curve preview, total duration, a four-phase breakdown table (duration and proportion per phase), and a segmented proportion bar. Action buttons:

  • ▶ Start — starts an instance directly from here. Triggers a haptic tap on confirmation. If the type is a substance and interaction warnings are enabled, an interaction check runs first.
  • Customize / Edit — opens the type editor
  • Hide — moves the type card to the collapsible Hidden section. The type still works normally; it just de-clutters the main list.
  • ✕ — deletes the type (custom only; confirmation required)
  • ★ / ☆ — toggles favorite. Favorited types appear first in the add-stopwatch sheet.
  • ↗ — (substance types only) opens the substance's PsychonautWiki article in the system browser.

If any instances of a type are currently running, a small count badge (pill in the type's color) appears next to the name.

Hiding a type. Tap Hide on any type card to remove it from its section without deleting it. The type keeps all its settings and can still be started from the Add Stopwatch sheet. A collapsible Hidden (n) section appears at the bottom of the list. Expand it to see all hidden types; each card shows an Unhide button that returns the type to its original section. Hidden state is persisted in AsyncStorage.

Graph

  • Plan overlay mode — controls how toggled-on plans are rendered on the Graph screen. Markers (default): one vertical start-time line per entry, labeled with plan and substance name — keeps the live signal clean. Curves: draws the full predicted curve for each plan entry at its type color, overlaid on the live graph for a side-by-side comparison of planned vs. live.

About

Attribution rows showing the data sources used in the bundled substance database. The first two rows are tappable links that open in the system browser:

  • PK data source — PsychonautWiki ↗ — opens https://psychonautwiki.org
  • Interaction data — TripSit combos ↗ — opens https://tripsit.me
  • Substances — 16 bundled profiles
  • Privacy & disclosure — View ↗ — opens the Disclosure screen (see below)

Disclosure screen

A full-screen page sheet summarising DoseAngel's privacy stance and harm reduction context. It contains three cards:

  • 100% offline · no data leaves your device — confirms that everything is stored locally and nothing is transmitted externally.
  • For harm reduction reference only — explains that PK data are population midpoints, individual responses vary significantly, and the app is not medical advice.
  • Interaction warnings — explains the TripSit-sourced interaction data and notes that the absence of a warning does not imply safety.

First launch: the Disclosure screen appears automatically on the first time the app is opened. A Don't show again checkbox (checked by default) is shown above the Continue button. If left checked, a flag is written to AsyncStorage and the screen will not appear automatically on subsequent launches. Unchecking it and tapping Continue dismisses the screen without writing the flag, so it will appear again next launch.

The screen can always be reopened via Settings → Privacy & disclosure → View ↗. In this context the checkbox is absent; a small Show on next launch link at the bottom clears the suppression flag so the screen will reappear automatically on the next cold start.

Disclaimer

A reminder that DoseAngel provides pharmacokinetic reference data for harm reduction purposes only — duration estimates are population midpoints, individual responses vary, and a medical professional should be consulted if in doubt.

5 Working with Plans

When adding or editing a plan entry, you set its timing via one of three anchor modes:

Start at

Pick the clock time at which the stopwatch begins. Offset = target time − now.

"I took it at 14:30."

Peak at

Pick when the peak phase begins. Start is back-calculated: start = peak − (onset + comeup).

"I want full effect by 19:00."

Comedown

Pick a target value (% of peak) and the clock time it should be reached during offset. Start is solved via binary search.

"25% by 17:00."

The drum-roll time picker

All time-editing flows (plan entries and running stopwatch start times) use a drum-roll scroll picker with three columns: day, hour, and minute.

  • Day column — scrolls from −3d to +3d, with "Today" in the centre. Labels: "−3d", "−2d", "−1d", "Today", "+1d", "+2d", "+3d".
  • Hour column — 0–23, circular (scrolling past 23 wraps back to 0 and vice versa).
  • Minute column — 0–59, circular.

The selected item is the large centred value; adjacent items are shown smaller and faded. Scroll any column to snap to the desired value.

Keyboard editing: tap the selected (large) item in the hour or minute column to open a combined HHMM keyboard overlay. A four-digit display appears (HH MM). Each digit you type barrel-shifts from the right — typing 1 4 3 0 produces 14:30. Tap elsewhere, press Return, or press any non-digit key to commit and return to scroll mode. The overlay seeds from the current picker value so you start from where you left off.

Offset display

The offset from now is shown in a human-readable format that scales with duration:

  • < 1 hour: +45m
  • 1–24 hours: +2h 15m (minutes omitted if exactly on the hour)
  • 1+ days: +1d 3h 30m (zero components omitted)
  • Past entries use a − prefix (e.g. −30m from now)

Timeline preview

A Start → Peak → End strip below the picker shows computed wall-clock milestones, updating live as you adjust the dials.

State model

All state lives in a single AppState managed by useReducer:

interface AppState {
  types:                   StopwatchType[];        // all type definitions (built-in + substance + custom)
  activeStopwatches:       ActiveStopwatch[];      // running instances
  favoriteTypeIds:         string[];               // ordered favorite IDs
  plans:                   Plan[];                 // saved plans
  showInteractionWarnings: boolean;                // toggle for blocking warning popups (default: true)
  showInteractionBadges:   boolean;                // toggle for ⚠ badges in type lists (default: true)
  planOverlayMode:         'markers' | 'curves';  // how toggled-on plans appear on the Live screen
}
State is persisted to AsyncStorage on every change and rehydrated on launch. Built-in types (isBuiltIn: true) and substance types (isSubstance: true) are always restored from code at hydration and are never saved to AsyncStorage — only custom types are merged from storage. This means bundled data updates automatically when the app version is updated.

Graph component

Graph accepts two optional override props:

  • overrideEntries?: GraphEntry[] — when provided, renders these entries instead of the live active stopwatches. Pass [] to suppress all curves (Plan Mode does this so only markers are visible).
  • planMarkers?: PlanMarker[] — renders vertical start-time marker lines on the graph. Each marker shows a full-height colored line with a label chip ("Plan name · Substance name").
interface GraphEntry {
  type:      StopwatchType;
  startTime: number;  // absolute ms
}

interface PlanMarker {
  startTime: number;  // absolute ms
  label:     string;  // e.g. "Festival · Extended"
  color:     string;  // CSS/RN color string
}

Paths are built from 150 sample points, split at "now" into solid (past) and dashed (future) SVG <Path> elements.

PlannedEntry model

Plan entries store an absolute timestamp rather than a relative offset:

interface PlannedEntry {
  id:         string;
  typeId:     string;
  targetTime: number;  // absolute ms (was: offsetMinutes)
}

On hydration, older persisted entries with offsetMinutes are automatically migrated: targetTime = Date.now() + offsetMinutes × 60 000.

8 Limitations & Roadmap

Priority roadmap (next up)

  1. 1
    iOS App Store release

    Submit to App Store under Health & Fitness, age rating 17+. Prepare screenshots, metadata, and privacy disclosure (no data leaves the device).

  2. 2
    Notifications

    Notify the user when a plan entry's scheduled start time arrives. No phase-transition or threshold notifications. Opt-out available in Settings.

  3. 3
    Simplify type customization screen

    The current type editor exposes all parameters at once (onset/comeup/peak/offset durations, peak value, onset end fraction, curve shapes per phase), which is overwhelming for new users. Redesign it with a simple mode — a small set of high-level controls (e.g. total duration, intensity, speed of onset) that map to sensible underlying parameters automatically — and an advanced mode toggle for users who want full control. The live sparkline preview should remain prominent in both modes so changes are immediately visual.

  4. 4
    Rename "Stopwatches" to a better term

    The word "stopwatch" is technically accurate but feels clinical and unfamiliar in a harm reduction context. Find a term that better communicates "active substance I'm currently tracking" — candidates include Sessions, Active doses, Trackers, or Running. The rename would touch the tab label, screen titles, card copy, and all in-app references. Decision should be made alongside UX review of the overall information architecture.

  5. 5
    Onboarding & in-app guidance

    First-time users currently have no guidance beyond the disclosure screen. Add lightweight onboarding that explains the core loop (track → graph → plan) without being intrusive. Options to explore: a short walkthrough on first launch (skippable), contextual empty-state hints (e.g. "Tap + to start tracking your first substance"), tooltip overlays on key UI elements, and a persistent How it works page accessible from Settings. The goal is to make the superposition model and plan/graph features discoverable without requiring users to read external documentation.

  6. 6
    UI polish

    Improve overall visual appeal — typography, spacing, color usage, card design, and transitions. Goal is a polished, trustworthy feel appropriate for a safety-focused tool.

No history

Stopped stopwatches are simply removed — no log of what ran and when.

Potential future features

History log

Record of all past runs: start time, stop time, integrated area under the curve.

Type library sync

Backup and restore custom types across devices.

Emergency numbers by country

A dedicated screen showing the relevant emergency and poison-control numbers for your country. Searchable/filterable country dropdown. Numbers open the phone dialer directly. Intended as a quick-access safety reference — not a replacement for calling emergency services.

Reagent testing guide

Per-substance reference showing which reagents to use (Marquis, Mecke, Simon's, etc.) and the expected color reactions for a positive result. Helps users verify substance identity before consumption. Sourced from published harm-reduction references (DanceSafe, Reagent.co.uk).

Drag-to-reorder types

Allow users to reorder stopwatch types within each section (Custom, Built-in, Substances) via drag handles. Order is persisted across sessions. Currently disabled pending a reliable cross-session implementation.

Addiction support resources

A dedicated screen listing addiction help lines, apps, and consultation services — searchable by country. Entries include phone numbers (tap to dial), links to apps (e.g. SMART Recovery, Workit), and links to professional consultation services. Sourced from established harm reduction organisations (DanceSafe, The Loop, local health authorities). Presented as a non-judgmental, always-accessible resource — not gated behind any usage pattern.

Redosing frequency warnings

Warn when the same substance is being redosed unusually often or too many times, independent of the pairwise interaction checker. This targets tolerance buildup and cumulative toxicity risk from repeated dosing of a single substance, rather than combinations between different substances. Thresholds and exact framing (per-session count, minimum interval, rolling time window) still to be defined.

Add mephedrone (3-MMC / 4-MMC) as a built-in substance

Not currently in the built-in substance list in constants/substanceDB.ts, though it already appears in the combos.json interaction database. Add it as an isBuiltIn substance type with onset/comeup/peak/offset curve parameters sourced from published duration data. Users can already track it today as a custom type in the meantime.
